# 회의실 예약 시스템 기능명세서

## 1. 개요
- **서비스명**: 회의실 예약 시스템
- **목적**: 사내 회의실(회의실, 뒤주)의 주간 예약 일정을 관리하고, 팀별 미팅 참여자 및 참석 여부를 효율적으로 관리
- **사용 대상**: 사내 임직원

---

## 2. 주요 기능

### 2.1 주간 캘린더 뷰
- **월~금 주간 일정**을 한눈에 확인
- **시간대**: 08:00 ~ 20:00 (5분 단위)
- **회의실 필터**: 회의실 / 뒤주 / 전체
  - 기본 상태: 전체 예약 보기
  - **전체 보기 시 겹치는 예약**: 같은 시간대에 회의실과 뒤주 예약이 겹칠 경우 너비를 반반씩 나누어 함께 표시
- 예약 블록에 **프로젝트명**을 상단에 작게 표기
  - 프로젝트가 선택된 경우 팀명은 표시되지 않음
- 예약 블록에 **참석 확인 인원 수** 표기

### 2.2 예약 생성
- **예약 항목**:
  - 회의실 선택 (회의실 / 뒤주)
  - 날짜 선택
  - 시간 선택 (시작/종료, 5분 단위)
  - 사용 목적 입력
  - 팀 선택 (복수 선택 가능)
    - 전체, C레벨, FE, BE, APP, EVE
  - 프로젝트 선택
  - 참여자 선택/해제
  - 미팅 메모 입력 (최대 250자)
    - 메모 내 URL 자동 링크 변환
- **프로젝트 또는 팀명 중 하나만 선택해도 예약 가능**
- **반복 예약(벌크 등록)**:
  - 동일한 내용(팀, 프로젝트, 회의실, 시간, 참여자, 목적, 메모)으로 여러 날짜에 일괄 예약
  - 반복 범위: 시작일 ~ 종료일 선택
  - 반복 요일 선택 (예: 월~금, 월/수/금 등)
  - 등록 후 각 날짜는 **개별 예약으로 분리**되어 개별 수정/삭제 가능

### 2.3 예약 수정 및 삭제
- 기존 예약 클릭 시 상세보기 및 수정 가능
- 개별 예약만 수정/삭제 (벌크 등록 후에도 각 날짜별 독립적 관리)

### 2.4 팀 관리
- **팀 목록**: 전체, C레벨, FE, BE, APP, EVE
- 팀 선택 시 해당 팀의 **기본 참여자 자동 선택**:
  | 팀 | 기본 참여자 |
  |---|---|
  | 전체 | 김명훈, 박지호, 김재진, 최우석, 이창욱, 김민성, 류주헌, 김윤하, 임성수, 배유진, 박성헌 |
  | C레벨 | 김명훈, 박지호 |
  | FE | 김재진, 최우석, 이창욱 |
  | BE | 김민성, 류주헌, 박지호 |
  | APP | 박성헌 |
  | EVE | 김윤하, 임성수, 배유진 |
- **복수 팀 선택 가능**: 여러 팀을 동시에 선택하면 해당 팀들의 참여자가 모두 기본 선택됨

### 2.5 프로젝트 관리
- **기본 프로젝트**: 수콘분청, 포치타, 페이빌더
- 사용자가 직접 **프로젝트 추가 / 이름 수정 / 삭제** 가능
- 프로젝트는 예약 시 선택하여 사용

### 2.6 참여자 및 참석 관리
- **참여자 명단**: 김명훈, 박지호, 김재진, 최우석, 이창욱, 김민성, 류주헌, 김윤하, 임성수, 배유진, 박성헌
- 팀 선택 시 기본 참여자 자동 선택, 개별 추가/해제 가능
- **참석 확인**: 예약 완료 후 상세보기에서 개인별 참석 여부 체크
  - **온라인 참여** / **현장 참여** 중 선택
  - 체크 즉시 저장 (자동 반영)
  - 캘린더 뷰에 참석 확인한 인원 수 표시

### 2.7 실시간 동기화
- Lovable Cloud 연동으로 예약 데이터 실시간 동기화
- 다중 사용자 동시 접근 시 데이터 일관성 유지

## 3. 데이터 모델

### 3.1 예약(Reservation)
| 필드 | 설명 |
|------|------|
| id | 예약 고유 ID |
| room | 회의실 (회의실 / 뒤주) |
| date | 예약 날짜 (YYYY-MM-DD) |
| start_hour, start_minute | 시작 시간 |
| end_hour, end_minute | 종료 시간 |
| purpose | 사용 목적 |
| teams | 선택된 팀 배열 |
| project_id | 연결된 프로젝트 ID |
| participants | 참여자 이름 배열 |
| attendance | 참석 상태 {이름: 'online' | 'onsite'} |
| notes | 미팅 메모 |

### 3.2 프로젝트(Project)
| 필드 | 설명 |
|------|------|
| id | 프로젝트 고유 ID |
| name | 프로젝트명 |

## 4. 기술 스택
- **Frontend**: React 18, Vite 5, TypeScript 5, Tailwind CSS v3
- **Backend**: Lovable Cloud
- **Database**: PostgreSQL
- **UI Library**: shadcn/ui

---

## 5. 화면 구성

### 5.1 메인 화면 (주간 캘린더)
- 상단: 주간 날짜 표시, 주 이동 버튼, 새 예약 버튼, 회의실 필터
- 중앙: 월~금 시간대별 그리드
- 예약 블록: 프로젝트명(상단) + 목적 + 참석 인원

### 5.2 예약 등록/수정 폼
- 회의실, 날짜, 시간, 목적, 팀, 프로젝트, 참여자, 메모 입력
- 반복 예약: 날짜 범위 + 요일 선택 UI

### 5.3 예약 상세보기
- 예약 정보 표시
- 참여자별 참석 확인 (온라인/현장) 체크박스
- 메모 내 URL 자동 링크 렌더링
- 수정/삭제 버튼

### 5.4 프로젝트 관리 다이얼로그
- 프로젝트 추가, 이름 수정, 삭제

---

## 6. 제약사항 및 규칙
- 예약 시간: 08:00 ~ 20:00 (5분 단위)
- 미팅 메모: 최대 250자
- 프로젝트 또는 팀 중 하나 이상 선택 시 예약 가능
- 같은 회의실 동일 시간대 중복 예약 방지
- 반복 예약 등록 후 각 날짜는 독립적인 예약으로 관리
